Proof it was worth doing

What the audit changed

Ten categories yielded a concrete fix or a new control in PCC's own code — several were the exact "a green that's actually red" disease PCC exists to kill, caught in PCC itself.

FAKE-GREEN

"Env is git-ignored" was a false promise

The rules told every worker secrets were safe from git — but .env wasn't actually ignored. Now it is, and the fix rides into every project PCC spawns.

SILENT FAIL

Security failures vanished without a trace

A failed revoke of build permission was silently dropped — you'd never know. Now every such failure lands in a durable log the health check surfaces in plain language.

FALSE ALL-CLEAR

A "smoke alarm" flashed green while checking nothing

The bloat signal reported "within limits" when its config was broken and it had scanned zero files. Now it honestly says "unknown."

UNTESTED GATE

The rule-book validator had no tests

The check that blocks every commit if a decision record is malformed — itself a constitutional file — had zero tests, and an old note claimed it was "proven" from a one-time manual check. Now 8 tests lock it in.

LINCHPIN

The branch-protection check was silently dead

The verifier the whole "un-bypassable" guarantee rests on could never actually run (the tool it needed was off the system path). Fixed, and wired into the release gate so a disabled backstop now fails loudly.

State, data & recovery

State & data integrityFixed / built
Atomic writes, journaled paired writes, and fail-closed recovery keep durable state truthful under crashes and corruption. Closed two constitutional files that had no test.
Backup & recoveryTested — holds
A restore aborts before touching any live file if the snapshot is incomplete — a bad backup can't half-overwrite good work.
Residue Taking a restore point is a habit, not enforced — but your git commits are the real, enforced safety net.
Recovery, rollback & damage-repairTested — holds
Interrupted writes self-heal; corruption serves the last-good copy and blocks further writes; it never fabricates a healthy state.

The behavior you're trusting — and the failures you won't tolerate

Confirm these match what you actually want, or correct them. This is the intent the whole system is measured against.

What PCC is meant to do

  • Reduce babysitting — you set direction; it runs and stops for real decisions.
  • Never fake "done" — a claim is proven, or plainly marked unproven.
  • Honest surfaces — every green means a real check said so; unknown reads as unknown.
  • Protect your data — nothing is lost; damage is recoverable; you're never locked in.

The failure modes it must never allow

  • Silent data loss — corruption must fail visibly, never overwrite good state.
  • A green that's actually red — no painted-over unknown or stale result.
  • Work marked done that isn't — no self-certified "verified" without independent proof.
  • Acting without authority — no build/no destructive action without your explicit grant.
